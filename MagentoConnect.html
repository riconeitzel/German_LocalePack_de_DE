Deutsches Sprachpaket für Magento Commerce

Fehler gefunden?

Sollte das Sprachpaket nicht wie gewohnt funktionieren, bitte Caches löschen und neu einloggen. Geht es danach immernoch nicht, bitte das Sprachpaket nochmal deinstallieren und wieder neu installieren.

Fehler in Version 1.7.0.x Seit 1.7.0.2.3 ist ein Fehler behoben, der beim Konvertieren des Connect 1.0 in einer 2.0 Paket entstanden ist. Dabei wurden einige Dateien nicht korrekt als Datei entpackt sondern als leere Ordner angelegt, die den Dateinamen als Ordnernamen tragen.

Bei einem Update der Extension wird dies zwangsläufig zur Problemen kommen, da diese defekten Ordner nicht gelöscht werden, eine Datei mit dem gleichen Namen aber nicht daneben gelegt werden darf.

Treten bei Ihnen diese Probleme auf, deinstallieren Sie das Sprachpaket im Magento Connect Manager, löschen Sie bitte anschließend folgende Ordner sofern sie vorhanden sind: /app/design/adminhtml/default/default/locale/de_DE/translate.csv, /app/design/adminhtml/default/default/layout/localepackede.xml, app/design/frontend/default/default/locale/de_DE/translate.csv, /app/design/adminhtml/base/default/locale/de_DE/translate.csv, /app/design/adminhtml/base/default/layout/localepackede.xml, app/design/frontend/base/default/locale/de_DE/translate.csv und installieren Sie das Sprachpaket erneut.

Fehler können auf GitHub gemeldet werden https://github.com/riconeitzel/German_LocalePack_de_DE/issues
